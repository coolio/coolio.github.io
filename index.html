See? Look at that, it's easy! But how's it work? Let's take a peek into the parts of my DSL:

  • cool.io.connection - Create a new connection class. Give your connection a name, then use the block to define callbacks for various events.
  • on_connect, on_close, on_read - These are callbacks fired when events occur, specifically when we get a new connection to our echo server, when a connection closes, and when data is read from the network.
  • cool.io.server - This creates a new server that listens on the given host and port. When someone connects to our server, a new connection of the given type will be created. In the case of our echo server, we'll create new :echo_server_connections.
  • cool.io.attach - This attaches the newly created server to the default cool.io event loop. This means the event loop will listen for events from the server. There's also cool.io.detach which will stop the event loop from monitoring events from our server.
  • cool.io.run - This runs the default cool.io event loop. The event loop will block, waiting for events. In the case of our echo server, it will wait for incoming connections, and once it's received them, it will wait for the client to send data, the echo it back to the client.
